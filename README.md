# BIT 研讨室预约助手

> 北京理工大学研讨室预约系统的油猴脚本增强版，彻底解决原网页加载慢、操作繁琐的问题。

## 为什么要写这个？

北理工研讨室预约系统（`stu.bit.edu.cn`）基于学校统一门户框架开发，存在以下问题：

- **加载极慢**：每次打开页面需要经历多次重定向和鉴权，等待时间长
- **交互繁琐**：原生界面操作路径深，预约一个场地需要点击多个页面
- **体验差**：页面布局陈旧，信息密度低，每次预约都要重新填写个人信息

本项目通过油猴脚本完全接管原网页，在保持与后端 API 正常通信的前提下，提供一套全新的前端界面，做到：

- 页面打开即可用，无需等待原框架的层层跳转
- 所有操作集中在一个页面完成
- 自动保存个人信息，下次预约直接复用

## 功能特性

- **场地浏览**：展示所有可预约研讨室，支持按日期、管理单位筛选
- **场地详情**：查看场地地址、容纳人数、设备设施、预约须知
- **时间段查询**：实时查询指定日期的可预约时间段，直观显示可用/已占用状态
- **在线预约**：填写预约表单，支持多时间段合并预约
- **我的预约**：查看历史预约记录及审核状态
- **撤回预约**：支持撤回待审核预约、删除草稿
- **信息记忆**：自动保存姓名、电话等常用信息，下次预约无需重复填写

## 安装使用

### 方式一：直接安装（推荐）

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 浏览器扩展
2. 前往 [Releases 页面](https://github.com/yang-kun-long/bit-reserve-userscript/releases/latest) 下载 `bit-reserve-userscript.user.js`，用 Tampermonkey 打开安装
3. 访问 [研讨室预约系统](http://stu.bit.edu.cn/xsfw/sys/cdyyapp/)，脚本自动生效

> 使用前需先登录北理工统一身份认证，脚本自动复用浏览器 Cookie。

### 方式二：开发模式

```bash
# 安装依赖
npm install

# 启动开发服务器（http://localhost:3000）
npm run dev
```

启动后在 Tampermonkey 中安装开发版脚本，支持热更新。

## 构建

```bash
npm run build
```

构建产物位于 `dist/bit-reserve-userscript.user.js`，可直接安装。

## 技术实现

### 技术栈

- **Vue 3**：前端框架，组件化开发
- **Vite**：构建工具
- **vite-plugin-monkey**：油猴脚本打包插件，处理 GM API 和外部依赖

### 关键机制

**预热流程（Preheat）**

原系统有 WAF（Web 应用防火墙）保护，直接调用预约接口会被拦截。脚本启动时会自动模拟浏览器行为，依次请求以下接口完成"预热"：

1. 访问研讨室系统首页，获取 `APPID`
2. `getAppConfig`：获取应用配置
3. `indexmenu.getAppConfig`：获取菜单配置，提取 `role_id`
4. `changeAppRole`：切换到对应角色
5. `setXgCommonAppRole`：设置通用角色
6. `i18n`：加载国际化配置

完成预热后，WAF 会认为这是正常的浏览器会话，后续的预约请求才能正常通过。

**Cookie 复用**

脚本运行在已登录的浏览器环境中，直接复用浏览器的 Cookie，无需用户再次输入账号密码。这也是为什么使用前需要先在浏览器中登录北理工统一身份认证。

**信息本地持久化**

姓名、电话等预约必填信息通过 `GM_setValue` / `GM_getValue` 存储在 Tampermonkey 的本地存储中，跨页面、跨会话持久保存，下次打开直接复用。

## 注意事项

- **需要先登录**：脚本依赖浏览器 Cookie 鉴权，使用前请确保已在浏览器中登录北理工统一身份认证
- **仅限校内系统**：本脚本仅适用于北理工研讨室预约系统，不适用于其他学校
- **预约时间限制**：系统通常只开放未来 7 天内的预约，脚本已做相应限制
- **审核机制**：提交预约后需等待管理员审核，可在"我的预约"中查看审核状态

## 常见问题

**Q：打开页面后脚本没有生效？**

确认 Tampermonkey 已启用，且脚本已安装并处于开启状态。检查脚本的匹配规则是否包含 `stu.bit.edu.cn`。

**Q：提交预约时提示"请先登录"或请求失败？**

Cookie 已过期，请重新登录北理工统一身份认证后刷新页面。

**Q：查询时间段一直加载中？**

可能是网络问题，或者 WAF 预热失败。尝试刷新页面重新加载脚本。

**Q：开发模式下热更新不生效？**

确认 Tampermonkey 中安装的是开发版脚本（`http://localhost:3000/` 地址），而非 `dist` 目录下的生产版本。
